What changed

  • Section labels: 🔥 🎵 ⚡ 🍳 ✨ → Feather-style line icons (quote marks, music notes, star, bowl, lotus)
  • Card-label text: "TODAY'S FIRE" → "TODAY'S QUOTE" (vibe-specific flavour removed to let icons carry the meaning; some users found the per-vibe labels confusing)
  • Breathing button: ✦ glyph → circular "breath ring" SVG — subtle and matches the breathing metaphor
  • Anthem mini-tile: 🎵 emoji → stroke music-note icon with vibe accent colour
  • Nav: ☀ sun character → stroked sun SVG; settings gear → stroked gear SVG. Matches the calendar icon we already have.
  • Kept emoji: food emojis inside fuel rows (🐟 🍗 🥣) — these are genuinely informative (you can tell the food type at a glance) and not decorative. Same for the vibe-selection emojis on onboarding.

Trade-off: the current design is warmer and more expressive — emojis convey emotion fast. Line icons feel more premium/clinical. For a fertility companion, the warmer version may actually be the right call — emoji overload on every label is the issue, not any single emoji. A middle-ground option: keep the 💛 friend-note emoji and vibe-colour dots, but swap the 8 section-label emojis for icons.

Recommendation: the middle-ground — icons for functional section labels, emojis reserved for decorative/emotional moments (friend-note heart, streak seedling, end-of-cycle orb).

Settings dropdown — reference patterns

Five options, adapted from real apps. Current app uses option B. Pick the one that feels right.
A · iOS SETTINGS
JOURNEY ›
Flat list row with trailing chevron. Tap takes you to a new screen (not expand). Best when sections have lots of content — avoids long scrolls. Used in iOS Settings, Messenger, Spotify.
B · CURRENT — INLINE EXPAND
Tinted button with chevron. Tap expands in place. Works for short sections. Used in Stripe Dashboard, Linear settings, Notion toggles.
C · MATERIAL EXPANSION
Journey personalisation ⌄
Full-weight heading (not mono-case), elevated card, larger chevron. Feels substantial, announces the section. Used in Google Material apps, GitHub settings. Heavier visual weight than current.
D · NOTION TRIANGLE
▶ JOURNEY
Leading triangle that rotates. No background, no border — very minimal. Best for dense UIs with many toggles (like Notion). Can feel too utilitarian for a consumer wellness app.
E · PREVIEW SUMMARY ★
Section header doubles as a summary. Useful for user state (shows what's currently set), collapsed by default. Used in Calm, Headspace, Duolingo settings. Tap opens the section.
F · DIVIDER + ARROW
JOURNEY PERSONALISATION +
Horizontal rule separator + plus/minus icon. Zero chrome, very minimal. Used in Medium profile settings, Substack. Feels like reading, not configuring.
My recommendation: E (preview summary).

Your sections are mostly about user state ("vibe: fierce", "treatment: IVF", "notifications: 8am"). A summary-header does two jobs at once: tells you what's in the section AND invites you to tap. It also makes the settings screen feel more like a profile than a config panel — which fits the emotional tone of the app.

If you want to keep the current pattern (B) but polish it: bump the header padding a bit more, add a subtle drop-shadow on hover, and animate the chevron rotate more visibly (0.3s with easing).
