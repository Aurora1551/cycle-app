How Reflections Are Stored
Current: Single reflection saved to localStorage as cycle_reflection — overwritten each cycle.

Proposed: Array of reflections saved as cycle_reflections — one per completed cycle. Each entry stores:
cycle_reflections = [
  {
    text: "That even on the hardest days...",
    treatment: "IVF",
    cycleDays: 18,
    date: "2026-03-20",
    vibe: "fierce"
  },
  {
    text: "I am not alone in this...",
    treatment: "IUI",
    cycleDays: 28,
    date: "2026-01-15",
    vibe: "nurturing"
  }
]
Recommendation
Use all three options together:

• Option A (full-screen moment) — shows once when starting a new cycle, if a reflection exists. Emotional, intentional, sets the tone.
• Option B (card on Day 1) — NOT needed if Option A exists, would feel redundant.
• Option C (Journey tab) — always visible. "Past Cycles" section at the bottom of Journey tab shows all completed cycles with their reflections. This is the permanent home.

So the journey is: End of cycle → write reflection → start new cycle → see reflection (Option A) → dismiss → Day 1. And all reflections are always browsable in the Journey tab (Option C).
